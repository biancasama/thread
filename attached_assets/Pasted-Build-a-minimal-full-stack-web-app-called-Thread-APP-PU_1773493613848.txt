Build a minimal full-stack web app called "Thread".

APP PURPOSE

Thread is an AI-powered external working memory tool that helps users recover their train of thought after interruptions, multitasking, or context switching.

This is a productivity and cognitive load support tool for general users.
It is NOT a medical product, not a therapy app, and it must not provide medical advice or diagnostic guidance.

The app should help users:
- capture a thought
- structure it into a meaningful thread
- recover context later
- understand what they were doing
- know what to do next

CORE PRODUCT IDEA

People often lose their train of thought while working.
Instead of saving raw notes only, Thread should use Gemini to transform short text or voice inputs into structured memory units called thought threads.

Each thought thread should contain:
- thread title
- goal
- current step
- important context
- next actions
- priority
- timestamp

Later, when the user clicks "Where was I?", the app should reconstruct the most relevant recent thread and display:
- what the user was working on
- the last known step
- suggested next actions

TECH STACK

Please use the fastest stack for a hackathon MVP.
Preferred:
- React frontend
- Node/Express backend if needed
- clean modern CSS or Tailwind if convenient
- Gemini API integration
- local state or local storage for persistence

Do not add:
- auth
- database
- payments
- collaboration
- unnecessary complexity

MVP FEATURES

1. HEADER
- App name: Thread
- Tagline: Recover your train of thought

2. THOUGHT INPUT PANEL
- Large text area where user can type a thought
- Capture button
- Optional voice input button if fast and simple
- Helper text like: "Capture what you're thinking, planning, or trying to do."

3. GEMINI PARSING
When the user submits a thought, send it to Gemini and extract structured JSON in this format:

{
  "thread_title": string,
  "goal": string,
  "current_step": string,
  "important_context": string,
  "next_actions": [string],
  "priority": "low" | "medium" | "high"
}

Use a prompt like:
"You are an AI that reconstructs human thought threads. Given a short text or speech transcript, infer the thread title, goal, current step, important context, next actions, and priority. Return valid JSON only."

Ensure JSON parsing is robust.
If the model output is malformed, fail gracefully.

4. ACTIVE THREAD PANEL
After parsing, display the latest thread in a clean card with:
- title
- goal
- current step
- important context
- next actions
- priority

5. RECOVER CONTEXT PANEL
Add a prominent button:
"Where was I?"

When clicked, show a recovery summary based on the most recent thread, like:
- You were working on: [thread title]
- Goal: [goal]
- Last known step: [current step]
- Next actions:
  1. ...
  2. ...
  3. ...

6. THREAD HISTORY
Show previous captured threads in a simple list or card column.

7. DEMO MODE / FALLBACK
Add a button called "Load Demo Thread".
This should populate the app with a hardcoded thread so that the app can still be demonstrated even if the API fails.

Use this exact demo data:
- thread_title: Hackathon Project - Thread
- goal: Build an AI external working memory assistant
- current_step: Preparing demo pitch
- important_context: The user is finalizing the hackathon demo
- next_actions:
  1. Write 15-second hook
  2. Prepare demo artifacts
  3. Record submission video
- priority: high
